# 🛒 Smart Shopping List — Документация и Руководство

Это современное веб-приложение (PWA/Telegram WebApp) для управления списками покупок с использованием искусственного интеллекта для автоматической категоризации товаров и глубокой вложенностью элементов.

---

## 📖 1. Руководство пользователя

### Вход и регистрация
*   При первом запуске необходимо создать аккаунт.
*   **Главный администратор:** `vasko8508@gmail.com` / Пароль: `123123`.

### Работа со списками
*   **Создание:** Нажмите кнопку `+` в нижнем меню.
*   **Закрепление:** Нажмите иконку "булавки", чтобы список всегда был сверху.
*   **Редактирование:** Иконка карандаша позволяет переименовать список.

### Управление товарами (Внутри списка)
*   **Добавление:** Кнопка "Добавить" открывает форму. 
*   **Группировка:** Вы можете переключить режим на "Группу". Группы позволяют создавать вложенные наборы (например, "Ингредиенты для пиццы").
*   **Интеллект:** При добавлении товара Gemini AI автоматически определит его категорию (Молочные продукты, Фрукты и т.д.).
*   **Действия:**
    *   `+` рядом с товаром — добавить под-товар (создать вложенность).
    *   `Разделить (Split)` — если у вас 10 шт. одного товара, кнопка разделит их на 10 отдельных строк для поштучной отметки.

### Совместный доступ
*   Нажмите иконку "Поделиться" в шапке списка.
*   **По ссылке:** Скопируйте ссылку. Любой человек сможет увидеть список (только чтение).
*   **Пользователи:** Выберите зарегистрированного пользователя из списка. Вы можете дать ему права "Только просмотр" или "Редактирование".

## 🏗 2. Техническая архитектура и файлы

Приложение построено на **React 19** с использованием **Tailwind CSS** и **Lucide Icons**.

### Основные файлы в корне:
1.  `App.tsx` — **"Мозг" приложения**. Управляет глобальным состоянием:
    *   Авторизация пользователя.
    *   Переключение между экранами (`activeView`).
    *   Загрузка/сохранение данных в `localStorage`.
    *   Обработка модальных окон (удаление, создание списка).
2.  `types.ts` — Описание структур данных. Если вы хотите изменить поля товара или пользователя, это делается здесь.
3.  `index.tsx` — Точка входа, инициализирующая React.
4.  `metadata.json` — Настройки разрешений и мета-данные приложения.

### Сервисы:
*   `services/geminiService.ts` — Интеграция с **Google Gemini API**. Отвечает за:
    *   `categorizeItem`: Автоматическое определение категории товара по названию.

### Компоненты (`components/`):
*   `AuthView.tsx` — Экран логина и регистрации. Работает с базой данных пользователей в `localStorage`.
*   `HomeView.tsx` — Главная страница со всеми списками.
*   `ListDetailView.tsx` — Самый сложный компонент. Управляет рекурсивным рендерингом товаров, вложенностью, формами добавления и логикой совместного доступа.
*   `StatsView.tsx` — Визуализация расходов с помощью библиотеки **Recharts**.
*   `ProfileView.tsx` — Настройки текущего пользователя (смена имени, пароля).
*   `AdminView.tsx` — Панель для главного администратора (управление правами других пользователей).

## 🔄 3. Взаимосвязи (Data Flow)

1.  **Пользователь** нажимает на кнопку в `App.tsx`.
2.  `App.tsx` меняет состояние `activeView`, и на экране отображается нужный **Component**.
3.  Если это `ListDetailView`, он получает объект `list` и функцию `onUpdateList`.
4.  При добавлении товара вызывается `geminiService.ts`, который возвращает категорию.
5.  Обновленный список отправляется обратно в `App.tsx`, который сохраняет его в `localStorage` (или в будущем отправит на бэкенд).
6.  Все изменения мгновенно отражаются в `StatsView.tsx`, так как он использует те же данные для построения графиков.

## 🗄 4. База данных (PostgreSQL)

В каталоге `init-db/` подготовлены скрипты для миграции с `localStorage` на реальный сервер:
*   `01_schema.sql` — создает таблицы, поддерживающие древовидную структуру (через `parent_id` в товарах).
*   `02_indexes_and_perms.sql` — оптимизирует скорость поиска.
*   `03_initial_data.sql` — создает первого админа.
